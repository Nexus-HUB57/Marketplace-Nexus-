**1. Por que LGPD + IA é crítico**

LGPD (Lei Geral de Proteção de Dados, Lei 13.709/2018) regulamenta qualquer operação com dados pessoais no Brasil.

Quando você roda um agente IA que:
- Coleta nome, e-mail, telefone → **dado pessoal**
- Analisa comportamento de compra → **dado pessoal**
- Faz perfil de personalidade → **dado pessoal sensível**
- Cruza dados de fontes diferentes → **tratamento de dados**

Tudo isso está sob LGPD. E a ANPD (Autoridade Nacional de Proteção de Dados) tem poder de:
- Advertência
- Multa de até 2% do faturamento (limitada a R$ 50M por infração)
- Bloqueio do banco de dados
- Suspensão da operação

**Multa média em 2024-2025**: R$ 4.2M.

**3. Os 7 princípios da LGPD que todo operador deve conhecer**

Rafael explica os 7 princípios:

1. **Finalidade** — você só pode usar o dado para o propósito declarado. Se coletou para "newsletter", não pode usar para "marketing multinível" sem novo consentimento.
2. **Necessidade** — colete apenas o mínimo. Não peça CPF se não precisa.
3. **Transparência** — o titular tem direito de saber o que você faz com os dados.
4. **Segurança** — proteja os dados com medidas técnicas razoáveis.
5. **Prevenção** — tome medidas ANTES de incidentes.
6. **Não discriminação** — IA não pode ser usada para discriminar.
7. **Responsabilização** — você é responsável, mesmo se delegou para terceiros.

**4. Case 1 — Multa de R$ 14M (varejista)**

Uma varejista brasileira, em 2024, foi multada em R$ 14M por usar agente de IA que:
- Coletava dados de clientes sem consentimento explícito
- Inferia perfil psicológico a partir de compras
- Vendia esses perfis para terceiros (parceiros de marketing)

**Erro principal**: o consentimento era genérico ("autorizo receber comunicações"). Não cobria inferência de perfil.

**Lição**: consentimento específico > consentimento genérico.

**5. Case 2 — Multa de R$ 1.8M (afiliado)**

Um afiliado Nexus (não vamos dar nome), em 2025, foi multado em R$ 1.8M por:
- Importar lista de leads de uma empresa terceira (sem verificar consentimento)
- Rodar agente que mandava mensagens no WhatsApp sem opt-in
- Não ter política de privacidade no site
- Não responder a 3 pedidos de titular (clientes pedindo para deletar dados)

**Erro principal**: o afiliado não sabia que **dados herdados têm compliance herdado**. Se a lista original não tinha consentimento LGPD, ele herda o problema.

**Lição**: due diligence em listas é obrigatório.

**6. As 5 violações mais comuns em agentes IA**

Camila lista o top 5 de violações que ela vê:

1. **Coleta sem consentimento** — agente pede nome+email sem explicar uso
2. **Retenção indefinida** — dados ficam armazenados "para sempre"
3. **Compartilhamento com terceiros** — sem aviso ao titular
4. **Falta de opt-out funcional** — não responder a "sair da lista"
5. **Logs excessivos** — agente registra tudo, incluindo dados sensíveis

**Regra de ouro**: se você não conseguir responder "por que estou coletando esse dado?", não colete.

---

**7. O "consentimento" em agentes**

Consentimento LGPD precisa ser:
- **Livre** (sem coerção)
- **Informado** (saber o que vai ser feito)
- **Inequívoco** (ação clara, não pré-marcado)

Para agentes, isso significa:
- **Opt-in** explícito (não opt-out)
- Linguagem clara sobre o que o agente vai fazer
- Opção de revogar a qualquer momento
- Registro do consentimento (timestamp + IP)

**Implementação técnica**: o agente precisa ter um campo `consent_given_at` no registro de cada usuário.

**8. Como implementar compliance na prática**

Rafael e Camila convergem em 5 passos:

```
PASSO 1: AUDITORIA
   Mapeie todos os dados pessoais que você coleta.
   Pergunte: "por que estou coletando isso?"

PASSO 2: POLÍTICA DE PRIVACIDADE
   Crie política clara, simples, em português.
   Publique no site. Atualize anualmente.

PASSO 3: CONSENTIMENTO
   Implemente opt-in em TODOS os pontos de coleta.
   Registre (log) o consentimento.

PASSO 4: DIREITOS DO TITULAR
   Implemente 5 direitos:
   - Acesso (ver dados)
   - Correção (corrigir dados)
   - Exclusão (deletar dados)
   - Portabilidade (exportar dados)
   - Revogação (cancelar consentimento)

PASSO 5: INCIDENTE
   Tenha um plano para incidente:
   - Quem é notificado (ANPD em 72h)
   - Como titular é avisado
   - Como sistemas são recuperados
```

**9. Checklist: você está em conformidade?**

Responda sim/não para cada:

- [ ] Tenho política de privacidade pública no site?
- [ ] Coleto consentimento explícito para cada uso?
- [ ] Mantenho registro de consentimento (log)?
- [ ] Respeito pedidos de exclusão em até 15 dias?
- [ ] Tenho plano de resposta a incidentes?
- [ ] Faço auditoria anual de dados?
- [ ] Limito acesso a dados por função (need-to-know)?
- [ ] Criptografo dados sensíveis em repouso e trânsito?
- [ ] Não compartilho dados com terceiros sem autorização?
- [ ] Tenho DPO ou consultor de privacidade responsável?

**Se respondeu "não" para 3+**: você está em risco. Priorize.
